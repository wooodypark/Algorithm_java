# Thinking-Driven Coding Practice
> "정답을 맞히는 것보다, 의도가 있는 코드를 쓰는 능력을 기른다."

## 🛠 나의 핵심 훈련 원칙
1. **코드보다 설계가 우선:** 구현 전 필요한 상태(변수), 자료구조, 복잡도를 먼저 정의한다.
2. **문제 재정의:** 문제를 그대로 받아들이지 않고 나만의 언어로 입력·출력·제약을 다시 쓴다.
3. **이유 있는 선택:** "왜 이 자료구조인가?"에 대해 최소 한 가지 이상의 근거를 든다.
4. **트레이드오프 인지:** 내가 선택한 방식의 장단점을 파악한다.

## 📈 Roadmap
- **Step 1:** DFS/BFS 기초 (백준 문제집) - 개인 훈련
- **Step 2:** 삼성 기출 (백준) - 스터디 기반 사고 훈련
- **Step 3:** 최신 기출 (코드트리) - 라이브 코딩 대비 실전 설계
- **Step 4:** 모의고사 (SWEA) - 실전 환경 및 환경 제약 적응
